# SMMplaner

Автоматизирует постинг в социальных сетях.

Считывает данные поста с таблицы и запускает постинг в трех социальных сетях: Вконтакте, Одноклассники и Telegramm.


## Запуск

Для работы с программой понадобиться создать проект в Google Clouds, создать приложение и подключить API к google sheets таблицам.
Создать таблицу, как в примере ниже:

| ID  | NAME  | DATE & TIME TO PUBLICATION  | PLATFORM  | TEXT | MEDIA LINK  | DATE & TIME TO DELETE  | STATUS  | ERROR DESCRIPTION  |
|:-:|---|---|---|---|---|---|---|---|
| 1 | Название поста  | Запланированное время публикации  | OK, VK  | Текст поста |Ссылка на изображение на Google диске| Запланированное время удаления  | Статус публикации  | Текст ошибки  |

Создать ТГ бота и ТГ канал, создать профиль и сообщество в ВК, создать профиль и сообщество в ОК.

Для запуска постинга вам понадобится Python третьей версии.

Скачайте код с GitHub. Установите зависимости:

```sh
pip install -r requirements.txt
```

Доступны три скрипта:

- `lauch_posting.py` - Публикует посты без указазанных даты и времени.
- `deleting_post_by_time.py` - Удаляет посты с указанными датой и временем удаления.
- `publishing_post_by_time.py` - Публикует посты с указанными датой и временем публикации.

## Переменные окружения

Часть настроек проекта берётся из переменных окружения. Чтобы их определить, создайте файл `.env` в корневой папке проекта и запишите туда данные в таком формате: `ПЕРЕМЕННАЯ=значение`.

Доступны следующие переменные:

- `SPREADSHEET_ID` -  ID google таблицы.
- `OK_APP_ID` - ID Приложения в одноклассниках.
- `OK_APP_SECRET_KEY` - Секретный ключ приложения в одноклассниках.
- `OK_ACCESS_TOKEN` - Токен пользователя в одноклассниках.
- `OK_GROUP_ID` - ID сообщества в одноклассниках.
- `POSTING_TELEGRAM_BOT_API_KEY` - Ключ телеграмм бота.
- `TELEGRAM_CHAT_ID` - ID Телеграмм сообщества.
- `VK_ACCESS_TOKEN` - Токен пользователя в VK.
- `VK_GROUP_ID` - ID группы VK.
